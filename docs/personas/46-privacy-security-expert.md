# Persona: Privacy / Security Expert

## Profile
- **Name:** Dr. Sarah Johnson
- **Age:** 42
- **Role:** Chief Information Security Officer (CISO)
- **Company Type:** Financial Services
- **Experience:** 16 years in cybersecurity and data privacy
- **Location:** London, UK

## Background
Sarah is responsible for protecting user data and ensuring security compliance. She evaluates websites for security vulnerabilities, data protection practices, and regulatory compliance (GDPR, PCI-DSS, etc.). She's particularly concerned about user trust and potential security breaches.

## Goals & Motivations
- Protect user data and privacy
- Ensure regulatory compliance
- Prevent security breaches and data leaks
- Build user trust through security
- Implement security best practices
- Stay ahead of evolving threats

## Pain Points & Challenges
- Developers who don't prioritize security
- Third-party scripts and integrations creating vulnerabilities
- Balancing security with user experience
- Keeping up with evolving regulations
- Legacy systems with security debt
- Limited security budget or resources
- User behavior that creates security risks

## What They Look For in a Website
- **HTTPS:** Full SSL/TLS encryption with valid certificates
- **Data Protection:** GDPR, CCPA, and regional privacy law compliance
- **Cookie Consent:** Proper consent management for tracking
- **Secure Forms:** Protection against injection attacks
- **Authentication Security:** Strong password requirements, MFA support
- **Authorization:** Proper access controls and permissions
- **Third-Party Vetting:** Secure, privacy-compliant third-party services
- **Data Minimization:** Collecting only necessary data
- **Privacy Policy:** Clear, comprehensive privacy documentation
- **Security Headers:** CSP, HSTS, X-Frame-Options, etc.
- **Vulnerability Management:** Regular security assessments
- **Incident Response:** Clear breach notification procedures
- **Data Retention:** Appropriate data retention and deletion policies

## How They Evaluate Personas
- **Data Sensitivity:** What sensitive data does each persona share?
- **Privacy Concerns:** How privacy-conscious is each persona?
- **Regulatory Coverage:** Which regulations apply to each persona?
- **Authentication Needs:** What authentication level is appropriate?
- **Trust Requirements:** What builds trust with each persona?
- **Breach Impact:** What's the impact of data breach per persona?
- **Consent Requirements:** What consent is needed for each persona?
- **Geographic Regulations:** What regional laws apply?
- **Risk Profile:** What security risks does each persona face?

## Key Quotes
> "Security and privacy aren't features—they're fundamental requirements."

> "Every third-party script is a potential security and privacy risk."

## Red Flags for This Persona
- Missing or invalid HTTPS
- No cookie consent mechanism
- Missing or inadequate privacy policy
- Collecting excessive user data
- Third-party trackers without consent
- No security headers
- Passwords transmitted insecurely
- SQL injection or XSS vulnerabilities
- No data encryption
- Missing GDPR compliance
- Insecure third-party integrations
- No incident response plan
- Weak authentication mechanisms
