## What They Look For in a Website
- **Clean Code:** Well-structured HTML, CSS, JavaScript
- **Performance:** Fast load times, optimized assets, lazy loading
- **Responsive Design:** Works seamlessly across all screen sizes
- **Accessibility:** Semantic HTML, ARIA, keyboard navigation
- **Browser Compatibility:** Works across modern browsers
- **Code Organization:** Modular, reusable components
- **Development Workflow:** Modern build tools, version control, CI/CD
- **CSS Architecture:** Maintainable stylesheets (BEM, utility-first, CSS-in-JS)
- **JavaScript Quality:** Modern ES6+, no memory leaks, proper error handling
- **Asset Optimization:** Compressed images, minified code, CDN usage
- **Testing:** Unit tests, integration tests, visual regression tests
- **Documentation:** Code comments, component documentation

## How They Evaluate Personas
- **Device Usage:** What devices and browsers do they use?
- **Screen Sizes:** What viewport sizes are most common?
- **Internet Speed:** Fast broadband or slow mobile connection?
- **Technical Proficiency:** Power users or non-technical?
- **Interaction Patterns:** How do they prefer to interact (mouse, touch, keyboard)?
- **Accessibility Needs:** Do they use assistive technologies?
- **Browser Preferences:** Chrome, Safari, Firefox, Edge?
- **JavaScript Enabled:** Can we assume JS is available?
- **Feature Expectations:** What UI patterns are they familiar with?

## Red Flags for This Persona
- Inline styles instead of proper CSS
- No responsive design or mobile consideration
- Poor performance (large unoptimized images, blocking scripts)
- Non-semantic HTML (div soup)
- No accessibility consideration
- jQuery or outdated libraries for simple tasks
- No build process or asset optimization
- Browser-specific code without fallbacks
- No error handling
- Tight coupling between components
- No version control or deployment process
