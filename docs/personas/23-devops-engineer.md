Rajesh manages infrastructure, deployment pipelines, and system reliability. He bridges development and operations, focusing on automation, monitoring, and continuous delivery. He's responsible for uptime, performance, and disaster recovery.

- Manual deployment processes prone to errors

## What They Look For in a Website
- **CI/CD Pipeline:** Automated testing, build, and deployment
- **Infrastructure as Code:** Terraform, CloudFormation, or similar
- **Containerization:** Docker containers for consistency
- **Orchestration:** Kubernetes or similar for container management
- **Monitoring:** Application and infrastructure monitoring (Prometheus, Datadog)
- **Logging:** Centralized logging (ELK stack, CloudWatch)
- **Alerting:** Automated alerts for issues
- **Scalability:** Auto-scaling capabilities
- **Disaster Recovery:** Backup and recovery procedures
- **Security:** SSL, firewall rules, vulnerability scanning
- **Performance:** CDN, caching, load balancing
- **Documentation:** Runbooks, architecture diagrams, deployment docs

## How They Evaluate Personas
- **Load Patterns:** When do different personas use the system?
- **Geographic Distribution:** Where are users located (CDN needs)?
- **Availability Requirements:** What uptime SLA does each persona expect?
- **Performance Expectations:** What latency is acceptable?
- **Scale:** How many concurrent users per persona?
- **Data Volume:** How much data do they generate?
- **Peak Usage:** When do they create load spikes?
- **Critical Paths:** Which journeys must always work?
- **Disaster Impact:** How does downtime affect each persona?

## Red Flags for This Persona
- Manual deployment process
- No monitoring or alerting
- No automated backups
- Single point of failure
- No load balancing or redundancy
- Hardcoded configuration
- No disaster recovery plan
- Missing SSL certificate or security config
- No CI/CD pipeline
- Poor logging or no centralized logs
- No infrastructure documentation
- No auto-scaling capabilities
