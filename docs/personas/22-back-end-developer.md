## What They Look For in a Website
- **Clean Architecture:** Well-organized codebase with separation of concerns
- **Database Design:** Normalized schema, proper indexing, query optimization
- **API Design:** RESTful or GraphQL APIs with proper versioning
- **Security:** Input validation, SQL injection prevention, XSS protection, authentication
- **Error Handling:** Comprehensive logging and error management
- **Testing:** Unit tests, integration tests, good code coverage
- **Performance:** Optimized queries, caching strategies, queue processing
- **Scalability:** Architecture that can handle growth
- **Documentation:** API documentation, code comments, architecture docs
- **Version Control:** Proper Git workflow with meaningful commits
- **CI/CD:** Automated testing and deployment pipelines
- **Monitoring:** Application performance monitoring, error tracking

## How They Evaluate Personas
- **Data Requirements:** What data does each persona need to access?
- **Transaction Volume:** How many requests will this persona generate?
- **Data Sensitivity:** What security/privacy requirements exist?
- **Integration Needs:** Do they need to connect with external systems?
- **Performance Expectations:** What response times are acceptable?
- **Concurrency:** How many simultaneous users per persona?
- **Data Retention:** How long must data be stored?
- **Compliance:** What regulatory requirements apply (GDPR, etc.)?
- **API Usage:** Will they access APIs programmatically?

## Red Flags for This Persona
- SQL injection vulnerabilities
- No input validation or sanitization
- Passwords stored in plain text
- Missing authentication/authorization
- No error logging or monitoring
- N+1 query problems
- No database indexes on queried fields
- Hardcoded credentials or secrets in code
- No automated testing
- Missing API versioning
- Poor separation of concerns
- No caching strategy
