# Persona: Product Manager

## Profile
- **Name:** Sophie Anderson
- **Age:** 33
- **Role:** Senior Product Manager
- **Company Type:** SaaS Technology Company
- **Experience:** 8 years in product management, previously software engineer
- **Location:** Cambridge, UK

## Background
Sophie owns the product roadmap and strategy for a digital product. She balances user needs, business goals, and technical feasibility. She's responsible for feature prioritization, user research, and product-market fit. She thinks in terms of outcomes, not outputs.

## Goals & Motivations
- Build products that solve real user problems
- Achieve product-market fit and user satisfaction
- Prioritize features based on impact and effort
- Make data-driven product decisions
- Balance user needs with business objectives
- Create roadmaps that align with company strategy

## Pain Points & Challenges
- Competing stakeholder demands and priorities
- Limited development resources
- Uncertainty about what users actually need
- Difficulty measuring feature success
- Pressure to ship features quickly vs. doing them right
- Technical debt that limits innovation
- Balancing new features with maintenance

## What They Look For in a Website
- **User-Centered Design:** Evidence of user research and testing
- **Feature Prioritization:** Clear focus on high-impact features
- **Product Metrics:** Tracking of engagement, activation, retention
- **User Feedback:** Mechanisms to collect user input
- **A/B Testing:** Capability to test hypotheses and iterate
- **Product Analytics:** Understanding of user behavior and feature usage
- **Onboarding Flow:** Clear user activation and aha moments
- **Value Delivery:** Features that solve real problems
- **Iteration Evidence:** Signs of continuous improvement
- **User Segmentation:** Different experiences for different user types
- **Product Vision:** Clear direction and purpose
- **Technical Excellence:** Reliable, performant, maintainable

## How They Evaluate Personas
- **Jobs to Be Done:** What job is the persona hiring the product to do?
- **Pain Points:** What specific problems do they face?
- **Outcomes Desired:** What does success look like for this persona?
- **Usage Patterns:** How frequently and in what context do they use the product?
- **Feature Needs:** Which features matter most to each persona?
- **Adoption Barriers:** What prevents them from using or adopting?
- **Value Metrics:** How do we measure value delivered to this persona?
- **Competitive Alternatives:** What else do they use or consider?
- **Willingness to Pay:** What's the pricing tolerance for each persona?
- **Growth Potential:** Which personas drive acquisition, retention, expansion?

## Technical Proficiency
- Expert: Product strategy, roadmapping, user research, feature prioritization
- Advanced: Product analytics, A/B testing, agile methodologies, UX principles
- Intermediate: SQL, basic coding, market analysis, competitive intelligence
- Tools: Jira, Productboard, Mixpanel, Amplitude, UserTesting, Figma

## Preferred Communication Channels
- Product roadmaps and strategy documents
- User research findings and insights
- Product analytics dashboards
- Feature specifications and user stories
- Competitive analysis and market research

## Key Quotes
> "Fall in love with the problem, not the solution."

> "Personas help me understand not just who users are, but why they do what they do."

> "A feature that doesn't solve a real user problem is just technical debt in the making."

## Red Flags for This Persona
- Features built without user validation
- No product metrics or analytics
- Missing user feedback mechanisms
- No clear product vision or strategy
- Kitchen-sink approach (too many features)
- Poor onboarding or user activation
- No A/B testing or experimentation
- Features that don't align with user needs
- No consideration of different user segments
- Building for stakeholders instead of users

## Website Evaluation Checklist
- [ ] Is there evidence of user research?
- [ ] Are product metrics and analytics in place?
- [ ] Do features solve real user problems?
- [ ] Is there a clear product vision?
- [ ] Are user feedback mechanisms available?
- [ ] Can the team run A/B tests?
- [ ] Is onboarding effective for new users?
- [ ] Are different user segments considered?
- [ ] Is the product roadmap user-driven?
- [ ] Are success metrics defined for features?
- [ ] Is there continuous iteration and improvement?
- [ ] Does the product deliver clear value?
- [ ] Are features prioritized by impact?
- [ ] Is technical performance excellent?
- [ ] Can users accomplish their jobs easily?
