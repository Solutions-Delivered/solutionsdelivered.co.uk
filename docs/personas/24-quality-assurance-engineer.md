# Persona: Quality Assurance Engineer

## Profile
- **Name:** Natalie Scott
- **Age:** 30
- **Role:** QA Lead
- **Company Type:** Software Company
- **Experience:** 7 years in quality assurance and testing
- **Location:** Bristol, UK

## Background
Natalie ensures software quality through comprehensive testing strategies. She combines manual testing with automation, focusing on functional, performance, and accessibility testing. She's detail-oriented and advocates for quality throughout the development process.

## Goals & Motivations
- Catch bugs before they reach production
- Improve software quality and reliability
- Automate repetitive testing tasks
- Advocate for quality in development process
- Ensure excellent user experience
- Reduce customer-reported issues

## Pain Points & Challenges
- Pressure to test quickly without compromising quality
- Limited time for comprehensive testing
- Developers who don't write testable code
- Missing or unclear requirements
- Regression issues from code changes
- Difficulty reproducing intermittent bugs
- Lack of test environments that match production

## What They Look For in a Website
- **Functional Testing:** All features work as specified
- **Regression Testing:** New changes don't break existing features
- **Cross-Browser Testing:** Consistent behavior across browsers
- **Responsive Testing:** Works on all device sizes
- **Performance Testing:** Load times, stress testing, scalability
- **Accessibility Testing:** WCAG compliance, screen reader support
- **Security Testing:** Input validation, authentication, authorization
- **Usability Testing:** Intuitive, user-friendly interface
- **Error Handling:** Graceful error messages and recovery
- **Edge Cases:** Unusual scenarios handled properly
- **Test Coverage:** Automated tests for critical paths
- **Documentation:** Clear acceptance criteria and test cases

## How They Evaluate Personas
- **Usage Patterns:** How will each persona use the system?
- **Critical Paths:** What workflows must work flawlessly?
- **Edge Cases:** What unusual scenarios might this persona encounter?
- **Device/Browser Mix:** What combinations need testing?
- **Error Scenarios:** What errors might they encounter?
- **Performance Needs:** What response times are acceptable?
- **Data Variations:** What data inputs will they use?
- **Accessibility Requirements:** What assistive technologies do they need?
- **Security Risks:** What security testing is needed per persona?

## Red Flags for This Persona
- No automated testing
- Bugs frequently found in production
- Broken functionality on certain browsers
- Poor error handling and validation
- Accessibility violations
- Slow performance under load
- Missing test environments
- Unclear acceptance criteria
- No regression testing process
- Features deployed without testing
- Security vulnerabilities
- Poor mobile experience
